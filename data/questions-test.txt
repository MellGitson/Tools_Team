## Questions de référence - Pydantic AI (Calibration)

### Happy Paths (6 questions) - Réponses claires dans le corpus

1. Comment définir un outil dans Pydantic AI ?
   Réponse attendue: Utiliser le décorateur @agent.tool ou la méthode agent.add_tool() avec une fonction Python et des types Pydantic

2. Quelle est la différence entre Agent et RunContext ?
   Réponse attendue: Agent est la configuration globale réutilisable du modèle et des outils, RunContext est le contexte d'exécution temporaire pour une invocation unique

3. Comment streamer une réponse avec Pydantic AI ?
   Réponse attendue: Utiliser agent.run_stream() avec async/await et boucler sur les chunks reçus

4. Quels sont les modèles LLM supportés par Pydantic AI ?
   Réponse attendue: OpenAI (gpt-4, gpt-3.5-turbo), Anthropic (claude-3), Groq, Ollama, Azure OpenAI, Google Vertex AI

5. Comment valider les réponses avec Pydantic AI ?
   Réponse attendue: Définir un response_schema typé en utilisant des modèles BaseModel Pydantic

6. Comment intégrer Pydantic AI avec FastAPI ?
   Réponse attendue: Créer un endpoint FastAPI qui utilise agent.run() pour traiter les questions et retourner les réponses

### Cas Tordus / Ambiguës (2 questions) - Plusieurs sections du corpus pourraient répondre

7. Comment gérer les erreurs dans Pydantic AI ?
   Réponse attendue: Plusieurs approches possibles - ValidationError pour Pydantic, LLMError pour les modèles, ToolError pour les outils, retry avec backoff exponentiel, logging

8. Comment optimiser les performances d'un agent Pydantic AI ?
   Réponse attendue: Réutiliser les agents (stateless), batch processing, caching, monitoring des appels, définir des timeouts
